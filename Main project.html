Digital Board Game Platform

Welcome to your colorful internship project page for a full-stack multiplayer board game platform. This design highlights both the frontend/backend workflow and the MERN stack path, plus an automatic demo board to show gameplay in motion.

Python + Flask Node.js + Express React + WebSockets MongoDB / MySQL RabbitMQ Messaging

Project Overview

Core Objective

Create a multiplayer portal featuring Chess, Checkers, secure server-side game logic, link-based matchmaking, and interactive chat.

Full-Stack Vision

Combine a responsive React frontend with a backend API, real-time sockets, and persistent database storage to support live matches and game history.

Automatic Gameplay

Include a demo mode that shows automatic piece movement. This can act as a live preview of game rules and match flow for users and mentors.

Tech Stack & Concepts

Frontend

  • React.js UI for game lobby and board rendering
  • HTML/CSS for vibrant responsive styling
  • WebSockets for real-time board and chat updates

Backend

  • Flask / Python for game validation and secure logic
  • Node.js + Express for API endpoints and matchmaking
  • Passport.js for authentication and session handling

Database / Messaging

  • MongoDB for user data and game state in MERN mode
  • MySQL for structured game records in Python mode
  • RabbitMQ for scalable in-game chat and match events

Frontend / Backend Process

1. User enters lobby

React loads the lobby page, displays active game rooms, and allows users to create or join matches.

2. Matchmaking request

The frontend sends a match request to the backend. The API pairs players based on game type, skill level, and availability.

3. Game state update

Moves are validated server-side with Flask/Node. WebSockets broadcast board updates and chat messages instantly to both players.

4. Chat + history

Chat messages flow through RabbitMQ, while completed games are saved to MongoDB or MySQL for replay and statistics.
